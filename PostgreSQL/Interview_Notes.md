# PostgreSQL Interview Notes

## Heap

Stores the actual table rows.

Heap is unordered, allowing fast inserts and updates.

---

## Pages

Heap is divided into fixed-size pages (8 KB).

The page is the smallest unit of disk I/O.

---

## B+Tree

Stores:

Key -> Heap Pointer

Supports fast lookups and efficient range scans.

---

## Query Planner

Chooses the lowest-cost execution plan.

Does not always use an index.

---

## MVCC

Updates create new row versions instead of overwriting.

Benefits

- Readers don't block writers.
- Writers don't block readers.

Tradeoff

Dead tuples require VACUUM.

---

## WAL

Append-only log.

Purpose

- Durability
- Crash Recovery
- Replication

Rule

WAL must reach disk before modified heap pages.

---

## Checkpoint

Flushes dirty pages from memory to disk.

Reduces crash recovery time.

---

## VACUUM

Removes dead tuples created by MVCC.

Prevents table and index bloat.

---

## Replication

Replica continuously replays WAL generated by the primary.

Primary handles writes.

Replicas handle reads.

---

## Partitioning

Splits one large table into smaller partitions.

Improves query performance and maintenance.

Not the same as sharding.
